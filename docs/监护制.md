# 监护制规则

## 概述

监护制是核电站操作中的核心安全规程。操作人执行关键操作时，监护人必须在旁全程监护，双方通过语音交互确认指令正确性，确保操作安全。

## 角色定义

| 角色 | 职责 | 识别方式 |
|------|------|---------|
| 操作人（ROAD） | 执行操作指令，读出设备九字码 | 目标检测 + 工位坐标匹配 |
| 监护人（LEADER） | 核对指令正确性，下达执行命令 | 目标检测 + 工位坐标匹配 |

## 正确操作路径

### 第一阶段：请求监护

```
操作人说出："请求监护"
    ↓
系统检测到"监护请求"意图（语音识别）
    或 操作人举手（姿态检测）
    ↓
进入 REQUESTING 状态
```

### 第二阶段：九字码复述

```
操作人：左手指向程序中的待执行指令
        右手鼠标点开操作控件
        读出设备九字码（如 "1ES013V"）
    ↓
监护人：通过程序核对当前指令正确
        指向操作控件
        复述九字码
        下达"可以执行"命令
    ↓
系统检测到设备码 + 确认意图
```

### 第三阶段：执行操作

```
操作人：按照程序指令操作设备
监护人：对操作过程全程监护
    ↓
双方检查设备状态与程序要求一致
    ↓
操作人：执行下一个操作
监护人：持续监护
```

### 第四阶段：核对确认

```
所有操作执行完毕
    ↓
操作人：核对设备状态
监护人：确认操作结果正确
    ↓
双方检查设备状态与预期一致
```

### 第五阶段：监护结束

```
操作人说出："实验结束"
    或 监护人离开超过 10 秒（距离 > 560px 持续 10秒）
    ↓
进入 IDLE 状态
```

## 状态机

```
IDLE ──→ REQUESTING ──→ BOUND ──→ IDLE
         (请求监护)     (监护中)    (结束)
```

### 状态转移条件

| 转移 | 触发条件 | 说明 |
|------|---------|------|
| IDLE → REQUESTING | 语音"监护请求" 或 举手 | 操作人发起监护请求 |
| REQUESTING → BOUND | 监护人到位（距离 ≤ 280px 持续 3秒） | 监护人走到操作人身旁 |
| BOUND → IDLE | 距离 > 560px 持续 10秒 | 人员离开超10秒 |
| BOUND → IDLE | 语音"实验结束" | 操作完成 |

## 距离检测

### 距离阈值

| 区间 | 距离 | 状态 | 说明 |
|------|------|------|------|
| 近距离 | ≤ 280px | 监护中 | 监护人在操作人身旁 |
| 中距离 | 280-560px | 接近中 | 监护人正在靠近 |
| 远距离 | > 560px | 未监护 | 监护人未到位或已离开 |

### 距离检测频率

- 每 30 帧检测一次（约 1 秒）
- 基于目标跟踪的中心点坐标计算欧氏距离

## 语音交互规则

### 关键意图

| 意图 | 关键词 | 触发动作 |
|------|--------|---------|
| 监护请求 | 请求监护、请监护、申请监护 | 启动监护流程 |
| 操作指令 | 设备码 + 动作词（开启/关闭/长按/核对） | 记录操作步骤 |
| 核对确认 | 核对、核实、验证 | 记录核对动作 |
| 确认 | 好、确认、没问题、收到 | 记录确认动作 |
| 实验结束 | 实验结束、试验结束、实验完成 | 结束监护流程 |

### 设备码识别

- 正则匹配：`1ES\w+|T1RPA\w+|LCO\w+|RPA\w+|SM3`
- 示例：1ES013V、T1RPA034、LCO306.6

## 举手检测

### 检测条件（满足任一即可）

- 手腕高于同侧肩膀 15px 以上
- 手肘高于同侧肩膀 10px 以上
- 手腕高于鼻子（高举过头）

### 确认机制

- 投票窗口：3 帧
- 投票阈值：3 帧中至少 2 帧检测到举手
- 冷却期：300 帧（约 10 秒）内不重复触发

## 配置参数

```yaml
supervision:
  bind_hold_sec: 3.0        # 监护人到位等待时间（秒）
  unbind_hold_sec: 10.0     # 人员离开容忍时间（秒）
  dist_close_px: 280        # 近距离阈值（像素）
  dist_near_px: 560         # 中距离阈值（像素）
  vote_window: 3            # 举手投票窗口（帧）
  vote_threshold: 2         # 举手投票阈值（帧）
  cooldown_frames: 300      # 举手冷却期（帧）
```

## 前端可视化

### 视频画面

- **绿色线**：监护人与操作人距离 ≤ 280px（监护中）
- **黄色线**：距离 280-560px（接近中）
- **红色线**：距离 > 560px（未监护）
- **灰色线**：非监护状态下，显示监护人与操作人的距离

### 事件面板

- 角色分配事件
- 举手检测事件
- 监护状态变化事件
- 距离监控事件（每 30 帧）

### 语音面板

- 实时语音转写
- 意图标签（监护请求、操作指令、确认等）
- 设备码高亮显示

## 与其他模块的交互

### 语音模块 → 监护状态机

- 语音模块发布 `VOICE_INTENT` 事件
- 监护状态机订阅并检测"监护请求"和"实验结束"意图

### MOT 模块 → 监护状态机

- 举手检测发布 `MOT_HAND_RAISED` 事件
- 监护状态机订阅并触发举手监护

### 监护状态机 → 评估模块

- 监护流程结束后，评估模块对整个流程进行评分
- 评分维度：九字码复述、操作执行、核对确认、监护完整性
